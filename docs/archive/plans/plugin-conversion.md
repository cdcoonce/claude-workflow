# Plan: Convert Presets to Claude Code Plugins

> Source PRD: GitLab #15 — "PRD: Convert presets to Claude Code plugins"
> Grill session: 20 decisions resolved in /grill-me

## Architectural decisions

Durable decisions that apply across all phases:

- **Plugin output structure**: `dist/<name>/` with `.claude-plugin/plugin.json`, `skills/`, `agents/`, `hooks/` (with `hooks.json` + `scripts/`), `settings.json`, `README.md` at root level
- **Source path variable**: All hooks and settings use `$CLAUDE_PLUGIN_ROOT` (replaces `$CLAUDE_PROJECT_DIR`)
- **Versioning**: Semver string in each preset's `manifest.json` (new `version` field), copied to `plugin.json` by build script
- **Marketplace**: `.claude-plugin/marketplace.json` at repo root, references `dist/<name>/` subdirectories
- **Build pipeline**: `build_preset.py` reads same `core/` + `presets/` sources, outputs plugin format instead of `.claude/` format
- **Validation**: `smoke_test.py` validates plugin structure (plugin.json, skill/agent integrity, hook refs, intra-skill links)
- **Agent role defaults**: Embedded directly in each agent's AGENT.md via `skills.add`/`skills.remove` (no separate `agent-role-defaults.json` in output; agents are fully self-contained)
- **CLAUDE.md**: Not generated by plugin. Each plugin's README.md includes a CLAUDE.md template for teams to copy.
- **Removed components**: `diff_preset.py`, `.template-version` generation, CLAUDE.md concatenation

Validation checks for the new format:
- `.claude-plugin/plugin.json` exists with required fields (name, version, description)
- Every directory in `skills/` has a `SKILL.md`
- Every directory in `agents/` has a valid `AGENT.md` (frontmatter with name, description, role)
- Agent skills.add references resolve to existing skills
- `hooks/hooks.json` references scripts that exist in `hooks/scripts/`
- Intra-skill markdown links resolve to existing files

Remove:
- `scripts/diff_preset.py`

Update:

- Smoke test runner entry point and CLI interface
