## Findings

### 1. Path Containment Bypass in Exclusion Processing

**File:** `scripts/build_preset.py:268-278`
**Confidence:** HIGH
**Severity:** High

**Vulnerable pattern:** The exclusion path containment check uses string `.startswith()`:

```python
excluded_path = (dist_path / exclusion).resolve()
if not str(excluded_path).startswith(str(dist_path.resolve())):
    ...continue...
shutil.rmtree(excluded_path)  # or .unlink()
```

String prefix comparison is an unreliable path containment check. If `dist_path` resolves to `/project/dist/preset`, a sibling directory `/project/dist/preset-other/` would also pass the `.startswith()` check. Symlink-based bypasses are also possible on systems that support them.

**Input source:** `manifest.get("exclude", [])` — loaded from `presets/*/manifest.json`, which is repo-controlled but authored by preset contributors.

**Impact:** An exclusion value crafted to resolve to a path that shares the dist_path prefix (but is outside it) could delete files outside the intended build directory via `shutil.rmtree()`.

**Recommended fix:** Replace string comparison with `Path.is_relative_to()` (Python 3.9+):

```python
excluded_path = (dist_path / exclusion).resolve()
if not excluded_path.is_relative_to(dist_path.resolve()):
    print(f"WARNING: exclusion '{exclusion}' resolves outside build directory, skipping")
    continue
```

### 2. Uncontained Path Resolution in Smoke Test Link Validation

**File:** `scripts/smoke_test.py:213-228`
**Confidence:** MEDIUM — Needs verification
**Severity:** Low

**Pattern:** Resolves relative link targets from SKILL.md files without checking containment:

```python
resolved = (skill_md.parent / link_target).resolve()
if not resolved.exists():
    result.errors.append(...)
```

**Input source:** Markdown link targets in SKILL.md files (preset-authored).

**Impact:** Limited — this only checks file existence and appends an error string. No file read/write/delete occurs. However, a malicious SKILL.md could probe for the existence of files outside the plugin directory. This is informational; the risk is path traversal as an oracle for file existence.

**Recommended fix:** Add containment check if defense-in-depth is desired:

```python
resolved = (skill_md.parent / link_target).resolve()
if not resolved.is_relative_to(dist_path.resolve()):
    result.errors.append(f"Skill link escapes plugin directory: {link_target}")
    continue
```
