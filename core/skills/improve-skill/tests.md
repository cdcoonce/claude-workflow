# Test Suite — improve-skill

| ID  | Scenario                                                                               | Expected Behavior                                                                                                                                                        | Result | Reason |
| --- | -------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ | ------ |
| T00 | Skill SKILL.md must be ≤100 lines                                                      | Claude counts lines; if >100, reports the violation and halts execution.                                                                                                 |        |        |
| T01 | Slug is provided via `/improve-skill <slug>` and matches `core/skills/{slug}/SKILL.md` | Resolves skill path correctly and records it in `skill_path`; proceeds to state file check.                                                                              |        |        |
| T02 | Slug provided does not match any skill in `core/skills/` or `presets/*/skills/`        | Aborts with: "Error: no skill found for slug "{slug}". Searched core/skills/{slug}/SKILL.md and presets/\*/skills/{slug}/SKILL.md."                                      |        |        |
| T03 | `/improve-skill` invoked with no slug argument                                         | Replies with usage instructions and stops — does not proceed to any phase.                                                                                               |        |        |
| T04 | State file exists with `status: completed` or `status: abandoned`                      | Notifies user, starts a new run using `-2` (or `-3`, etc.) suffix to avoid collision.                                                                                    |        |        |
| T05 | State file exists with `status: in_progress`                                           | Reports "Resuming {slug} at phase {current_phase} (best score: {best_score}%)." and jumps directly to that phase.                                                        |        |        |
| T06 | Phase 2 Grill — AskUserQuestion tool is available                                      | All interview questions are posed using AskUserQuestion tool with selectable options; no plain-text numbered Q&A is used.                                                |        |        |
| T07 | Phase 2 Grill — AskUserQuestion tool is NOT available                                  | Falls back gracefully to numbered text Q&A (1–5 questions in sequence); does not error or skip questions.                                                                |        |        |
| T08 | Phase 2 Grill — no tests.md exists yet                                                 | Claude reads the skill's SKILL.md and infers recommended test cases, presenting them as selectable options or suggestions before asking user for additions.              |        |        |
| T09 | Phase 2 Grill — tests.md already exists with N rows                                    | Shows a one-line summary of each existing test; asks what harder or missing cases to add; appends new rows only — never removes or renumbers existing rows.              |        |        |
| T10 | Phase 2 Grill — T00 row placement                                                      | T00 ("Skill SKILL.md must be ≤100 lines") is always the first data row in any newly created tests.md; never duplicated if suite already contains it.                     |        |        |
| T11 | Phase 3 Baseline — QA Tester agent scores the original skill                           | Baseline score is recorded in the state file under `baseline_score` and as iteration 0 in the Scores table; `best_score` is initialized to this value.                   |        |        |
| T12 | Phase 4 Iterate — Skill Writer rewrites, QA Tester rescores, loop repeats              | Skill Writer and QA Tester execute as a team across multiple iterations (not a single pass); loop continues until target pass rate is met or max iterations is reached.  |        |        |
| T13 | Phase 4 Iterate — two consecutive iterations with no score improvement                 | Stall detected: strategy agent is invoked to propose a rewrite strategy before the next Skill Writer iteration begins.                                                   |        |        |
| T14 | Phase 4 Iterate — iteration produces a score lower than the current best               | Best version is preserved; state file retains `best_score` and `best_iteration` from the previous peak; regression is logged.                                            |        |        |
| T15 | Phase 5 Finalize — improvement run completes                                           | A PR is filed, the state file `status` is set to `completed`, state file is archived to `docs/archive/skill-improve/`, and a score report (baseline → best) is produced. |        |        |
